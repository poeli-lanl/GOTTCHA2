# Genomic Origin Through Taxonomic CHAllenge (GOTTCHA2)

GOTTCHA2 is a **gene-independent, signature-based** metagenomic taxonomic profiler designed to reduce false discoveries while remaining practical to run on a workstation/laptop. It maps reads to precomputed **unique signature fragments** and reports abundance from **signature coverage / depth** rather than marker genes.

> **GOTTCHA v1 databases are NOT compatible with GOTTCHA2**.

## What’s new

Recent releases (v2.3.0) added a few major improvements:

- **BAM-first workflow**: alignment parsing and downstream processing now use sorted/indexed **BAM** rather than SAM for performance and scalability.
- **Identity / SNI updates**: consensus identity is computed using a majority-rule approach, and **SNI-score is computed from consensus identity** (replacing the earlier read-weighted identity as the primary SNI metric).
- **CLI refactor**: clearer subcommands, cleaner structure, and faster extraction.
- **Backward compatibility**: The legacy gottcha2.py entry point remains available for older workflows, but is frozen at v2.2.3 and will not receive further updates. For runs that produce SAM output, use `gottcha2 sam2bam` to convert SAM files generated by any GOTTCHA2 version into a sorted/indexed BAM for downstream steps.

See the **GitHub Releases** page for full change logs.

## Installation

### Option A: Conda (recommended)

```bash
conda install -c bioconda gottcha2
```

### Option B: Install from source

```bash
# external tool `minimap2` and `samtools` need to be installed separately
git clone https://github.com/poeli/GOTTCHA2
cd GOTTCHA2
python -m pip install .
# dev install:
python -m pip install -e .
```

Confirm the install:

```bash
gottcha2 version
gottcha2 profile --help
```

### Prebuilt databases

Prebuilt GOTTCHA2 databases are hosted here (may take a while to download):

A typical database bundle includes:

- `gottcha_db.<level>.fna.mmi` (minimap2 index; **required** for `profile -i ...`)
- `gottcha_db.<level>.tax.tsv` (taxonomy mapping; can be auto-discovered or passed via `--taxInfo`)
- `gottcha_db.<level>.stats` (signature length / genome size stats; used for coverage and abundance estimates)

### Download helper

The CLI includes a `download` subcommand; run this to see current options:

```bash
gottcha2 download --help
```

(If you prefer, you can always download a database bundle directly from the URLs above.)

## Quick start

### 1) Profile Illumina paired-end reads

```bash
gottcha2 profile \
  -d /path/to/gottcha_db.species.fna \
  -i sample_R1.fastq.gz sample_R2.fastq.gz \
  -t 8 -o out -p sample
```

### 2) Profile Illumina single-end reads

```bash
gottcha2 profile -d /path/to/gottcha_db.species.fna -i sample.fastq.gz -t 8
```

### 3) Profile Oxford Nanopore (ONT) reads

ONT mode expects **a single input file** and enables read preprocessing + different defaults (error rate / identity thresholds):

```bash
gottcha2 profile \
  -d /path/to/gottcha_db.species.fna \
  -i ont_reads.fastq.gz \
  --nanopore \
  -t 8
```


### 4) Re-run profiling from an existing BAM (skip mapping)

If you already have a **sorted + indexed** BAM from a prior run, you can re-run aggregation with different cutoffs without remapping:

```bash
gottcha2 profile \
  -b sample.gottcha_species.bam \
  -d /path/to/gottcha_db.species.fna \
  -Mc 0.01 -Mr 10 -mi 0.95 \
  -t 8
```

## Command overview

GOTTCHA2 uses a subcommand-style CLI:

```text
gottcha2 <command> [options]
```

Commands:

- `profile`  — map reads (FASTQ/FASTA) and profile taxa, or profile from an existing BAM
- `extract`  — extract reads for one or more taxa (shorthand for `profile --extract ... --extractOnly`)
- `sam2bam`  — convert a GOTTCHA2 SAM (legacy) to sorted/indexed BAM
- `download` — download a database bundle (see `--help`)
- `version`  — print version

## Profiling

### Key concepts

Metagenomic samples were taxonomically profiled by mapping sequencing reads directly to a database of precomputed, taxon-specific signature fragments. For each genome, GOTTCHA2 consolidates alignments across the genome’s signature space to compute coverage and depth statistics and an ANI-inspired metric called the signature nucleotide identity score (SNI-score). In practice, read-to-signature alignments are combined to derive mapping and coverage features, which are then used to estimate the SNI-score for each genome. Genome-level results are subsequently aggregated to higher taxonomic ranks, yielding abundance estimates and summary statistics across multiple levels of the taxonomy.

### Handling Oxford Nanopore Technologies (ONT) datasets

Because signature sequences vary in length and GOTTCHA2 reports only the single best match per query, the pipeline first converts the input ONT read file (FASTA or FASTQ, optionally gzipped) into a temporary FASTA file of fixed-length chunks to make long reads easier to map: each read is split into **non-overlapping 150 bp segments** (headers look like `>READID|chunk=N`, with an optional prefix), and any leftover tail shorter than 150 bp is dropped. After mapping these chunks, a postprocessing step cleans the SAM by grouping chunk alignments back by their original read ID (everything before `|chunk=`) and extracting a taxonomic ID from the reference name; it then keeps only the alignments matching the **most frequent taxid** per original read, discarding inconsistent chunks, and marks the first retained alignment for each read with a `ZC:i:1` tag before writing the filtered SAM output.

### Dealing with your "Acession-of-Interest"

These function are used when you want to **track or control reads that align to a specific set of signature fragments or accessions**, such as a list of plasmids, spike-ins, or any “targets of interest” you care about. You can provide that target list with `--accList`, a file containing one accession or the sequence ID per line. During the mapping/analysis step—when the pipeline examines read alignments and can tell which reference accession each read hit—`--accListAction` determines how those hits are handled: use `filter_out` to **remove** reads that map to any accession in the list (e.g., to exclude plasmid/contaminant reads from downstream profiling), `filter_in` to **keep only** reads that map to the listed accessions (e.g., to extract plasmid-associated reads for follow-up analysis), or `report_only` to **leave the dataset unchanged** while still reporting how many reads matched the list at the column `AOI_READ_COUNT` at the full report.

## Read extraction

GOTTCHA2 supports extracting multiple taxa simultaneously. Taxa can be specified using NCBI tax IDs or names, separated by commas (,) in a single string, or listed in a text file prefixed with an @ symbol (e.g., `-e @taxa_list.txt`). For example, option `-e "666,562"` will extract reads mapped to "Vibrio cholerae" and "Escherichia coli". It's equivalent to `-e "Vibrio cholerae,Escherichia coli"` or `-e "666,Escherichia coli"`

### Example usages

Extract reads that map to specific taxa (IDs or names):

```bash
# extract reads mapping to taxid 666
gottcha2 extract \
  -b sample.gottcha_species.bam \
  -e 666
```

Extract reads using differnt parameters:

```bash
# extract reads mapping to taxid 666, with min matching identity 0.9 and min matching fraction 0.9
gottcha2 extract \
  -b sample.gottcha_species.bam \
  -mi 0.9 -mf 0.9 \
  -e 666
```

Multiple taxa can be given as a comma-separated list, or via a file prefixed by `@`:

```bash
gottcha2 extract -b sample.gottcha_species.bam -e "1234,5678"
gottcha2 extract -b sample.gottcha_species.bam -e "@taxids.txt"
```

Limit reads and choose output format with `:N:FORMAT`:

```bash
# up to 1000 reads per taxon, FASTQ output
gottcha2 extract -b sample.gottcha_species.bam -e "@taxids.txt:1000:fastq"
```

Extract up to 20 representative sequences for each profiled taxon using:

```bash
gottcha2 profile -b sample.gottcha_species.bam --extractFullRef
```

### Extracted read record format

Each extracted record is written as a FASTA or FASTQ entry consisting of:

1. a header line beginning with `>` for FASTA or `@` for FASTQ that encodes metadata, followed by
2. the read sequence on the next line
3. (optional quality score for FASTQ format)

#### Header template

```text
>{READ_NAME}{MATE}|{REFERENCE}:{START}..{END} LEVEL={LEVEL} NAME={NAME} TAXID={TAXID} AOI={AOI} MG={MG} MI={MI} MF={MF}
```

#### Field definitions:

* **`READ_NAME`**: The query/read identifier.
* **`MATE`**: Paired-end mate suffix (only for paired-end reads):
  * `.1` if this alignment is from read 1
  * `.2` if this alignment is from read 2
  * empty for unpaired reads
* **`REFERENCE`**: Reference sequence name (`acc|start|end|taxid|`)
* **`START`..`END`**: The reference interval for the mapped region (1-based)
* **`LEVEL`**: Taxonomic rank/level of the extracted taxon
* **`NAME`**: Extracted taxon
* **`TAXID`**: Extracted taxonomy ID
* **`AOI`**: Acession-of-interest flag
* **`MI`**: Mapping identity
* **`MF`**: Mapping fraction
* **`MG`**: Alignment length

#### Example

```text
>read123.1|chrA|1|300|GCF10000:10..120 LEVEL=species NAME=Escherichia_coli TAXID=562 AOI=False MI=98.65 MF=0.99 MG=148
ACGT...
```

## Output files

By default, outputs go to `--outdir` (default: current directory) and use `--prefix` derived from the input.

Typical outputs:

- `*.tsv` / `*.csv` / `*.biom` — **summary** at the reporting level
- `*.full.tsv` — full, unfiltered report with per-taxon metrics + filter notes
- `*.lineage.tsv` — lineage + abundance table
- `*.mpa.tsv` — MetaPhlAn-style output (only if `--mpa`)
- `*.extract.fasta` / `*.extract.fastq` — extracted reads (only if `--extract`)
- `*.gottcha_<level>.bam` (+ `.bai`) — alignment BAM (can be reused for re-profiling)

## Thresholds and filtering

Most numeric cutoffs default to `0` (disabled) unless explicitly set. Filtered taxa are still present in the **full** report (`*.full.tsv`) with the reason recorded in `NOTE`.

Use `--noCutoff` to disable taxonomic profiling cutoffs: equivalent to `-Mc 0 -Mr 0 -Ml 0 -Mz 0 -ss 0,0,0` (alignment thresholds will remain applied).

### Parameters

Read alignment thresholds:

- `-mi, --matchIdentity <FLOAT>`  
  Minimum alignment identity (0.0-1.0) required to be considier a valid match. Default: `0.95` (short reads), `0.85` (nanopore).

- `-mf, --matchFraction <FLOAT>`  
  Minimum fraction (0.0-1.0) of the read or signature fragment required to be considered a valid match. Default: `0.95` (short reads), `0.85` (nanopore).

- `-mg, --matchLength <INT>`  
  Minimum alignment length (bp) required. Default: `100`.

Taxonomic profiling cutoffs:

- `-ss, --sniScore <FLOAT>[,<FLOAT>,<FLOAT>]`  
  SNI-score thresholds for aggregation (`other,species,strain`). Default: `0.9,0.95,0.99`.

- `-Mc, --minCov <FLOAT>`  
  Minimum signature coverage required to be considered valid in abundance calculation. Default: `0`.

- `-Mr, --minReads <INT>`  
  Minimum number of mapped reads required. Default: `0`.

- `-Ml, --minLen <INT>`  
  Minimum covered signature length required. Default: `0`.

- `-Mz, --maxZscore <FLOAT>`  
  Maximum estimated z-score for mapped-region depth distribution (coverage uniformity; lower is better). Default: `0` (disabled).

## Full report fields

The full report ([prefix].full.tsv) includes the complete set of metrics along with filtering flags and notes. The summary report ([prefix].tsv) contains only the first 11 columns, representing the qualified taxonomic results.

| Field Name             | Description |
| ---------------------- | ----------- |
| LEVEL                  | Taxonomic level (superkingdom → strain) |
| NAME                   | Taxon name |
| TAXID                  | NCBI Taxonomy ID |
| READ_COUNT             | Reads mapped to this taxon |
| TOTAL_BP_MAPPED        | Total aligned bases mapped to this taxon’s signatures |
| SNI_SCORE              | Signature nucleotide identity used for rank-level aggregation/filtering |
| COVERED_SIG_LEN        | Total length of covered signature regions |
| BEST_SIG_COV           | Highest signature coverage among rolled-up strain members |
| DEPTH                  | Depth of coverage (`TOTAL_BP_MAPPED / TOTAL_SIG_LEN`) |
| REL_ABUNDANCE_GC       | Relative abundance normalized from genomic-content estimate (`GENOMIC_CONTENT_EST`) |
| REL_ABUNDANCE          | Relative abundance from the field selected by `--relAbu` |
| PARENT_NAME            | Parent taxon name |
| PARENT_TAXID           | Parent NCBI taxid |
| AOI_READ_COUNT         | Reads mapped to accessions listed in `--accList` (when enabled) |
| TOTAL_READ_LEN         | Total aligned read length |
| TOTAL_BP_MISMATCH      | Total mismatched bases across alignments |
| TOTAL_BP_INDEL         | Total inserted/deleted bases across alignments |
| READ_WT_SNI            | Read-weighted identity estimate (`1 - TOTAL_BP_MISMATCH / TOTAL_READ_LEN`) |
| CONSENSUS_SEQ_SNI      | Consensus-sequence identity estimate (`1 - CONSENSUS_DIFF / COVERED_SIG_LEN`) |
| SNI_CI95_LH            | 95% CI bounds (low/high) for identity estimate |
| SIG_COV                | Signature coverage (`COVERED_SIG_LEN / TOTAL_SIG_LEN`) |
| MAPPED_SIG_LEN         | Signature length with ≥1 mapped read |
| TOTAL_SIG_LEN          | Total signature length for this taxon |
| COVERED_SIG_DEPTH      | Depth over covered signature only (`TOTAL_BP_MAPPED / COVERED_SIG_LEN`) |
| COVERED_MAPPED_SIG_COV | Fraction of mapped signature that is covered (`COVERED_SIG_LEN / MAPPED_SIG_LEN`) |
| ZSCORE                 | Depth-distribution Z-score |
| GENOMIC_CONTENT_EST    | Genomic-content estimate (`DEPTH * GENOME_SIZE`) |
| ABUNDANCE              | Raw abundance value from `--relAbu` |
| REL_ABUNDANCE_DEPTH    | Relative abundance computed from `DEPTH` |
| SIG_LEVEL              | Taxonomic level of signatures used for mapping |
| GENOME_COUNT           | Number of reference genomes rolled up into this taxon |
| GENOME_SIZE            | Combined genome size used for GC estimate |
| NOTE                   | Filter / rollup notes (e.g., cutoff failures or rank-bias checks) |

## Troubleshooting

### BAM must be sorted and indexed / Legacy SAM files

If you provide `-b/--bam`, the BAM should be coordinate-sorted and indexed (`.bai`). For legacy SAM files, use:

```bash
gottcha2 sam2bam -i sample.sam -o sample.bam -t 8
```

### Missing taxonomy mapping

If auto-discovery can’t find the right `*.tax.tsv`, pass it explicitly:

```bash
gottcha2 profile --taxInfo /path/to/gottcha_db.species.fna.tax.tsv ...
```

### Identity / SNI notes

Starting v2.3.0, **SNI_SCORE is derived from consensus identity** computed via a majority-rule approach (the read-weighted identity is still reported but is no longer the primary SNI used for filtering/aggregation). If you are comparing results across versions, keep in mind the identity/SNI computation change.
